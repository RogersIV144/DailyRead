# Workflow Overview

## Scope and Trigger

- 这个 topic 的边界是什么？当前为什么需要先做 workflow mapping？
- 这次 mapping 聚焦的是哪个 track、时间窗口、应用场景或技术路线？

## Workflow and Lifecycle Map

- 从上游输入到下游应用，这个 topic 的端到端 workflow 是什么？
- 每个阶段的核心任务、关键 artifact、主要 bottleneck 分别是什么？
- 哪些环节是稳定主线，哪些环节是可选分支？

## Layered Stack

- 这一方向最合理的层次结构是什么？
- 每一层的职责边界、输入输出和相邻层接口分别是什么？
- 哪些术语是同义改写，哪些其实代表不同研究对象？

## System and Architecture Roles

- system 在每一层通常负责什么？
- architecture 或 hardware-visible support 在每一层可能负责什么？
- 同一个问题如果放在 system vs architecture 去做，目标函数和约束会有什么差别？

## Worth-Doing Filter

- 哪些环节最值得继续深挖，为什么？
- 哪些环节虽然热门，但当前更像 crowded optimization 而不是好切入点？
- 如果资源有限，优先进入哪 1 到 3 个子问题最合理？

## Anchor Readings

- 用哪些 anchor papers 或 survey materials 建立最小认知闭环？
- 哪些阅读适合先看，哪些适合在已有基础后再看？

## Next-Step Handoff

- 下一步应该接 `daily-discovery`、`venue-survey`、`paper-analyze`、`idea-synthesis` 还是 `report-writer`？
- 进入下游 skill 时，应沿用哪些 focus question、layer terminology 或 intervention hypothesis？
